Sincronizando ACLs HDFS e Permissões de Sentinela

Este tópico apresenta um plugin HDFS-Sentry que permite configurar a sincronização de privilégios do Sentry com ACLs HDFS para diretórios HDFS específicos.

Anteriormente, quando o Sentry era usado para proteger dados em Hive ou Impala, era difícil compartilhar com segurança os mesmos arquivos de dados HDFS com outros componentes, como Pig, MapReduce, Spark, cliente HDFS e assim por diante. Você tinha duas opções:
  • Você pode configurar a propriedade para todo o armazém da Hive para Colméia: colméiaE não permitir que outros componentes tenham acesso aos dados. Embora isso seja seguro, ele não permite o compartilhamento.
  • Use ACLs HDFS e sincronize os privilégios do Sentry e ACLs HDFS manualmente. Por exemplo, se um usuário tiver apenas o SentrySELECIONE Privilégio em uma tabela, esse usuário só deve poder ler os arquivos de dados do quadro e não gravar esses arquivos HDFS.

Introdução

Para resolver o problema acima mencionado, o CDH 5.3 introduz integração de permissões do Sentry e HDFS que manterão as ACLs HDFS automaticamente sincronizadas com os privilégios configurados com o Sentry. Este recurso oferece a maneira mais fácil de compartilhar dados entre o Hive, Impala e outros componentes, como MapReduce, Pig, e assim por diante, ao estabelecer permissões para esses dados com apenas um conjunto de regras através do Sentry. Ele mantém a habilidade de Hive e Impala para definir permissões em vistas, além de tabelas, enquanto o acesso a dados fora da Hive e Impala (por exemplo, a leitura de arquivos fora HDFS) exige permissões de tabela. As permissões de HDFS para alguns ou todos os arquivos que fazem parte das tabelas definidas no Metastore de Raiva agora serão controladas pelo Sentry.

Essa mudança consiste em três componentes:
  • Um plugin HDFS NameNode
  • Um plugin para metadióts Sentry-Hive
  • Um plugin do Sentry Service

Com a sincronização habilitada, o Sentry irá traduzir as permissões nas tabelas para a correspondente ACL HDFS correspondente nos arquivos da tabela subjacente no HDFS. Por exemplo, se um grupo de usuários for atribuído a uma função Sentry que tenha permissão SELECT em uma tabela específica, esse grupo de usuários também terá acesso de leitura aos arquivos HDFS que fazem parte dessa tabela. Quando você lista esses arquivos no HDFS, essa permissão será listada como ACL de HDFS.

Observe que, quando o Sentry foi habilitado , oColméia O usuário / grupo recebeu propriedade de todos os arquivos / diretórios no armazém da Colméia (/ Usuário / colméia / armazém). Assim, as permissões Sentry sincronizadas resultantes refletirão esse fato.

O mapeamento dos privilégios do Sentry para ACLs HDFS é o seguinte:

  • Privilégio SELECT -> acesso de leitura no arquivo.
  • Privilégio INSERT -> Acesso de gravação no arquivo.
  • TODOS os privilégios -> acesso de leitura e gravação no arquivo.
Importante:
  • Com a sincronização ativada, sua habilidade para definir permissões HDFS para esses arquivos está desabilitada. As permissões para esses arquivos particulares podem ser configuradas somente através do Sentry e, quando examinadas através do HDFS, essas permissões aparecem como ACLs HDFS. Um conjunto configurável de usuários, comoColméia e Impala, Terá acesso total aos arquivos automaticamente. Isso garante que um requisito-chave do uso do Sentry with Hive e Impala - dando acesso a esses processos para regular as permissões nos arquivos de dados subjacentes - seja encontrado automaticamente.

  • Tabelas que não estão associadas ao Sentry, ou seja, não têm nenhum usuário com privilégios do Sentry para acessá-los, manterão suas ACLs antigas.

  • Com a sincronização HDFS-Sentry habilitada, se o plugin NameNode não puder se comunicar com o Sentry Service por um determinado período de tempo (configurável pelo Sentry.authorization-provider.cache-stale-threshold.msPropriedade), as permissões para todos os diretórios nos prefixos do caminho gerenciado pelo Sentry, independentemente de esses caminhos de arquivo corresponderem aos objetos do armazém da Arroz, serão definidas paraColméia: colméia.

  • A sincronização do Sentry HDFS não suporta a HA da metáfora da raiva.

Pré-requisitos

  • CDH 5.3.0 (ou superior)
  • (Fortemente recomendado) Implementar autenticação Kerberos em seu cluster.
As seguintes condições também devem ser verdadeiras ao ativar a sincronização do Sentry-HDFS. O não cumprimento de qualquer um destes resultados resultará em erros de validação.
  • Você deve usar o serviço Sentry, e não a autorização baseada em arquivo de política.
  • É necessária a ativação de listas de controle de acesso ampliado HDFS (ACLs) .
  • Deve haver exatamente um serviço Sentry dependente do HDFS.
  • O serviço Sentry deve ter exatamente uma função do Sentry Server.
  • O serviço Sentry deve ter exatamente um serviço de colméia dependente.
  • O serviço Hive deve ter exatamente uma função Hive Metastore (ou seja, alta disponibilidade não deve ser ativada).

Ativando o plugin HDFS-Sentry usando o Cloudera Manager

  1. Vá para o serviço HDFS.
  2. Clique na guia Configuração .
  3. Selecione Escopo > HDFS (Service-Wide) .
  4. Selecione Categoria > Todos .
  5. Tipo Verifique as permissões do HDFS Na caixa de pesquisa.
  6. Selecione Verificar permissões de HDFS .
  7. Selecione Ativar sincronização do Sentry .
  8. Localize a propriedade Prefixes do caminho de sincronização do Sentry ou procure-a digitando seu nome na caixa de pesquisa.
  9. Edite a propriedade Prefixes de caminho de sincronização do Sentry para listar prefixos de caminho do HDFS onde as permissões do Sentry devem ser aplicadas. Vários prefixos de caminho HDFS podem ser especificados. Por padrão, essa propriedade aponta paraUsuário / colméia / armazémE deve sempre ser não-vazio. A sincronização de privilégios HDFS não ocorrerá para tabelas localizadas fora das regiões HDFS listadas aqui.
  10. Clique em Salvar alterações .
  11. Reinicie o cluster. Observe que pode levar mais dois minutos após o reinício do cluster para a sincronização de privilégios entrar em vigor.

Ativando o plugin HDFS-Sentry usando a linha de comando

Importante:
  • Você pode usar o Cloudera Manager ou as seguintes instruções de linha de comando para completar esta configuração.
  • Esta informação aplica-se especificamente ao CDH 5 .4 .x . Se você usa uma versão anterior do CDH, consulte a documentação daquela versão localizada na Cloudera Documentation .

Para habilitar os plugins do Sentry em um cluster não gerenciado, você deve permitir explicitamente o Hdfs Usuário para interagir com o Sentry e instalar os pacotes de plugins conforme descrito nas seções a seguir.

Permitindo o Hdfs Usuário para se conectar com o Sentry

Para um cluster não gerenciado, adicione Hdfs ao Sentry.service.allow.connect Propriedade em Sentry-site.xml. 
<Property>
    <Nome> sentry.service.allow.connect </ name>
    <Valor> impala, colméia, matiz, hdfs </ value>
</ Property>

Instalando o HDFS-Sentry Plugin

Nota: Instale o Repositório Cloudera

Antes de usar as instruções nesta página para instalar o pacote, instale o Cloudera Yum, Zypper/YaST ou AptoRepositório, e instale ou atualize o CDH 5 e verifique se ele está funcionando corretamente. Para obter instruções, consulte Instalando a última versão CDH 5 .

Use as instruções a seguir, dependendo do seu sistema operacional, para instalar o Sentry-hdfs-pluginpacote. O pacote deve ser instalado (no mínimo) nos seguintes hosts:
  • O host que executa o NomeNome e o Nome secundárioNode
  • O host que está executando o Hive Metastore
  • O host que executa o Sentry Service
SO Comando
Compatível com RHEL 
$ Sudo yum instala sentry-hdfs-plugin
SLES 
$ Sudo zypper instala sentry-hdfs-plugin
Ubuntu ou Debian 
$ Sudo apt-get install sentry-hdfs-plugin

Testando os Plugins de Sincronização do Sentry

As seguintes tarefas devem ajudá-lo a garantir que a sincronização do Sentry-HDFS tenha sido ativada e configurada corretamente:

Para uma pasta que foi ativada para o plugin, como o armazém da Arroz, tente acessar os arquivos nessa pasta, além de Hive e Impala. Para isso, você deve saber a que tabelas esses arquivos HDFS pertencem e as permissões do Sentry nessas tabelas. Tente visualizar ou modificar as configurações de permissões do Sentry através dessas tabelas usando uma das seguintes ferramentas:
  • (Recomendado) Aplicação de segurança de Hue
  • HiveServer2 CLI
  • Impala CLI
  • Acesse os arquivos da tabela diretamente no HDFS. Por exemplo:
    • Liste arquivos dentro da pasta e verifique se as permissões de arquivos exibidas no HDFS (incluindo ACLs) correspondem ao que foi configurado no Sentry.
    • Execute um trabalho MapReduce, Pig ou Spark que acessa esses arquivos. Escolha qualquer ferramenta além de HiveServer2 e Impala
